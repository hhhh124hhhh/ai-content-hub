### 2. 质量评估 📊

**评估维度**（100 分制）：

#### 实用性 (30%)
**评估项**：
- 是否包含具体使用场景（10 分）
- 是否提供详细的步骤说明（10 分）
- 是否有示例输出（10 分）

**评分标准**：
```
包含使用场景（10 分）：
✓ 有明确的应用场景（如"用于写博客文章"） - 10 分
✓ 有一般性描述（如"适用于写作"） - 5 分
✗ 无使用场景说明 - 0 分

详细步骤（10 分）：
✓ 分步骤说明清晰（1. 2. 3...） - 10 分
✓ 有一定的步骤说明 - 7 分
✓ 步骤说明不清晰 - 3 分
✗ 无步骤说明 - 0 分

示例输出（10 分）：
✓ 有实际输入输出示例 - 10 分
✓ 有格式化示例 - 7 分
✓ 有简单示例 - 3 分
✗ 无示例 - 0 分
```

#### 创新性 (25%)
**评估项**：
- 方法是否独特（10 分）
- 是否有新颖的角度（10 分）
- 是否突破了常见模式（5 分）

**评分标准**：
```
独特方法（10 分）：
✓ 非常独特，未见过的 - 10 分
✓ 比较独特 - 7 分
✓ 常见方法 - 3 分
✗ 非常常见 - 0 分

新颖角度（10 分）：
✓ 完全新颖的角度 - 10 分
✓ 较新颖的角度 - 7 分
✓ 常见角度 - 3 分
✗ 传统角度 - 0 分

突破模式（5 分）：
✓ 突破了常见模式 - 5 分
✓ 有一定创新 - 3 分
✓ 传统模式 - 0 分
```

#### 完整性 (20%)
**评估项**：
- 说明详细程度（8 分）
- 参数说明是否清晰（7 分）
- 是否有注意事项（5 分）

**评分标准**：
```
说明详细（8 分）：
✓ 非常详细（> 200 字） - 8 分
✓ 比较详细（100-200 字） - 5 分
✓ 一般详细（50-100 字） - 3 分
✓ 简单说明（< 50 字） - 1 分
✗ 无说明 - 0 分

参数清晰（7 分）：
✓ 所有参数都有说明 - 7 分
✓ 大部分参数有说明 - 5 分
✓ 少数参数有说明 - 3 分
✗ 无参数说明 - 0 分

注意事项（5 分）：
✓ 有注意事项说明 - 5 分
✓ 有简单的注意事项 - 3 分
✗ 无注意事项 - 0 分
```

#### 热度 (25%)
**数据来源**：
- 点赞数
- 转发数
- 评论数
- 引用数
- 收藏数

**评分标准**：
```
热度计算：
热度分 = (点赞 × 0.4 + 转发 × 0.3 + 评论 × 0.2 + 引用 × 0.1)

归一化（25 分）：
热度分_normalized = 热度分 / 100 × 25
```

**示例**：
```
点赞 500，转发 100，评论 50，引用 10：
热度分 = 500 × 0.4 + 100 × 0.3 + 50 × 0.2 + 10 × 0.1
       = 200 + 30 + 10 + 1
       = 241

热度分_normalized = 241 / 100 × 25 = 60.25 分

四舍五入：60 分
```
